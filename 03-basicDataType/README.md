# Java的数据类型 

## 内置数据类型
### byte
- byte数据类型是8位,有符号,以二进制补码表示的整数
- 范围 -128 -> +127
- 默认是0
- byte类型用在大型数组中节约空间,主要代替整数,因为byte占用空间只有int的四分之一

### short
- short数据类型是16位,有符号以二进制补码表示的整数
- 范围 -32768 -> +32767
- short也可以像byte那样节省空间,一个short变量是int型变量所占空间的二分之一
- 默认值是0

### int
- int数据类型是32位,有符号以二进制补码表示的整数
- 范围 -2147483648 -> 2147483647
- 默认值0
- 一般整型变量默认为int类型

### long
- long 数据类型是 64 位、有符号的以二进制补码表示的整数；

- 范围 -9,223,372,036,854,775,808 -> +9,223,372,036,854,775,807

- 默认值是 0L

- 这种类型主要使用在需要比较大整数的系统上

- ```java
  long a = 10000L
  ```

### float
- float数据类型是单精度,32位,符合IEEE 754标准的浮点数

- float在储存大型浮点数组的时候可以节省内存空间

- 默认值 0.0f

- 不能用来表示精确的值,如货币

- ```java
  float f = 234.5f
  ```

### double
- double数据类型是单精度,64位,符合IEEE 754标准的浮点数

- double在储存大型浮点数组的时候可以节省内存空间

- 默认值 0.0d

- 不能用来表示精确的值,如货币

- ```java
  double f = 234.5
  ```

### boolean:
- boolean数据类型表示一位的信息
- 只有true,false两个值
- 默认false

### char
- char是一个单一的16位Unicode字符

- 最小值\u0000

- 最大值\uffff

- char数据类型可以储存任何字符

- ```java
  char letter = 'A'
  ```

  

## 引用数据类型
- 对象,数组都是引用数据类型

- 所有引用类型的默认值都是null

- 一个引用变量可以用来引用任何与之兼容的类型

- ```java
  Site site = new Site("Runoob")
  ```

  


## 常量
- 常量用final修饰,一般常量名用全大写

  ```java
  final double PI = 3.1415926
  ```

- 字面量可以赋给任何内置类型的变量

  ```java
  byte a = 68;
  char a = 'A';
  ```

- byte int long short 都可以用十进制,16进制,8进制的方式表示

  ```java
  int decimal = 100;
  int octal = 0144;
  int hexa = 0x64;
  ```

- 字符串常量和字符常量都可以包含任意Unicode字符

  ```java
  char a = '\u0001';
  String a = "\u0001";
  ```

- 转义字符

## 自动类型转换

- 不能对boolean类型进行类型转换

- 不能把对象类型转换成不相关类的对象

- 把大容器的类型转换为小容器的类型必须使用强制类型转换

- 转换过程可能导致溢出或者损失精度

  ```java
  int i = 128;
  byte b = (byte)i;
  ```

  因为byte类型是8位,最大值为127,所以当int强制转换为byte类型时,值128时候就会导致溢出

  | 数值            | 原码                                     | 反码                                     | 补码                                |
  | --------------- | ---------------------------------------- | ---------------------------------------- | ----------------------------------- |
  | 正数            | 第一位符号位0                            | 和原码相同                               | 和原码相同                          |
  | 负数            | 第一位符号位1                            | 原码符号位不变,其他位取反                | 反码 + 1                            |
  | int i = 129     | 00000000 00000000 00000000 10000001 ---> | 00000000 00000000 00000000 10000001 ---> | 00000000 00000000 00000000 10000001 |
  | byte(i)  = -127 | 11111111                                 | <---  10000000                           | <---     10000001                   |

  计算机在运算的时候都是以补码的方式来运算的,所以上面int强转截取后补码为10000001

  

- 浮点数到整数转换时通过舍弃小数得到的,不是四舍五入

  ```java
  (int)23.7 ===> 23;
  (int)-45.89f ===> -45;
  ```
